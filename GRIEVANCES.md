- can't see share button in reports page
  - **Root Cause**: While padding was added to `.reports-wrap`, the `#view-root` container relies on `padding-bottom: var(--nav-height)` to clear the fixed bottom navigation bar. On some mobile devices or browser viewports, `100dvh` combined with system UI bars or scroll container nesting causes the bottom padding to be swallowed or insufficient, causing the buttons to still fall under the nav bar.
  - **Fix**: Added `env(safe-area-inset-bottom, 24px)` plus an additional `24px` buffer directly to the `#view-root`'s `padding-bottom`, and significantly increased `.reports-wrap`'s bottom padding to `120px` to guarantee the share buttons are completely un-obscured regardless of device edge.
- homepage budget tile and streak line are awkwardly spaced
  - **Root Cause**: The layout awkwardness wasn't just about the exact pixel margins; it was because the "Current Streak" line was raw text floating directly between two prominent `.card` containers (`.budget-card` and "Today's Activity" card). A naked text row sandwiched between two elevated glassmorphism cards feels visually disjointed and "awkward".
  - **Fix**: Wrapped the "Current Streak" line inside its own `.card` element. This perfectly harmonizes it with the rest of the dashboard layout, fixing the perceived awkward spacing by giving it the same structural weight and padding as surrounding items.
- daily baseline text in settings and streak freezes in achievements are not visible in dark mode
  - **Root Cause**: The text items were rendered inside raw `<button>` elements (e.g., `.settings-row` and `.theme-toggle`). In CSS, `<button>` elements do not inherit `color` from `body` by default; they use the user-agent's `buttontext` default (usually black). While `body` successfully flipped `--text-primary` to white (`#f0fdf4`) in dark mode, the buttons stubbornly remained black text. Against the darkened background (`var(--bg-card)`), black text became practically invisible.
  - **Fix**: Explicitly added `color: var(--text-primary);` to the `.settings-row` and `.theme-toggle` button classes in `style.css` so they inherit and react to the dark mode text color swaps properly.
